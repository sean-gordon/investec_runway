API Usage

How Gordon Uses Investec's API

Not a screen scraper. Not a CSV importer. Gordon uses Investec's Programmable Banking API for everything — real-time, authenticated, and secure.

GET

Transaction History — /za/pb/v1/accounts/:id/transactions

Polled every 60 seconds. New transactions are deduplicated into TimescaleDB hypertables, then batch-categorised by AI and fed into the actuarial burn-rate model.

GET

Account Balances — /za/pb/v1/accounts/:id/balance

Fetched in parallel across all linked accounts on every sync cycle. Aggregated into a single net position and used as the baseline for runway calculations.

GET

Account Listing — /za/pb/v1/accounts

Used on startup and settings refresh to enumerate all accessible accounts — Private Bank, Business Current, Savings, and Money Market — without hardcoding any account IDs.

POST

Inter-Account Transfers — /za/pb/v1/accounts/:id/transfermultiple

When runway drops below a user-configured threshold, the Runway Top-Up Worker can automatically transfer funds from savings to cheque. Requires explicit user consent in settings. Every transfer is logged.
